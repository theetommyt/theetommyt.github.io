What is an Empty State?

The empty state happens when there is no data to populate a display; it is sometimes known as the empty list experience. Outside of error conditions, the empty state typically happens for one of three reasons:

  1. The user has not created any data
  2. The user has cleared all data
  3. The user's request returned an empty set of results

The goal is to comfort, guide, or encourage users when there is no content to show. Users need to feel reassured that we are going to help them use the product and be successful in their tasks. The idea is to give the user something to do, by rewarding them for fulfilling the primary call to action of the interface: creating or clearing data.

The user has not created any data

Users will encounter an empty state any time they visit a view before there is data to populate it. This may be part of the first use experience, or it may be the result of creating new, blank, data elements. The empty state is an opportunity to educate the user, and encourage them to create something.

If the primary call to action in an empty state is the creation of new data, the interface should answer the following questions:

  • What will appear in this interface when data is available?
  • What is the purpose of this interface?
  • How do I add data?

The user has cleared all the data

There are cases where the primary call to action is the elimination of data. For example, when a user views a list of pending approvals, the goal is to get the list to zero. Other examples include reading all notifications, completion of all background tasks, or the emptying of an outbound message queue.

These sort of empty states can be used to reward the users, and encourage them to take the next appropriate action in a workflow. For example, if all the background processing on a set of reports is done, encourage them to read the reports. A user who is clearing data is a user who’s engaged with the product. If the next step in a workflow is obvious, keep the user engaged by showing them the next step: don’t make the user think about what they have to do next.

The user's request returned an empty set of results

When a user is searching for something, or filtering a set of results, there is always a chance that they won’t find what they’re looking for. These scenarios are amazing opportunities to infer what your customer intended to find and to make intelligent suggestions.

Possible options include: taking into account misspellings and similar searches; giving the closest matching result; and highlighting which search items don’t have any matches. The point is to keep from boxing in the user: give them something they might be able to work with or suggest an alternate path.
