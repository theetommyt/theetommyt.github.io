Changing Data

Dealing with user input

When the user needs to change data or input new data, the ideal interface is sophisticated as necessary, and no more. With Apollo, we balance the the needs of the user with the needs of the data, and bring focus to the elements which require attention.

Creating new data, and editing existing data, are two aspects of the same action. As much as possible, the user experience should be consistent. These actions generally fall into two categories: convenient, and disclosed.

Convenient Input

When the data in question is self-contained, or frequently changed, use convenient input. "Convenient" methods include inline editing, and always-visible forms for adding new records.

Disclosed Input

When the data in question is complex, or when multiple related items must be changed at one time, use disclosed input. With disclosed input and editing, the user interface to edit, or create new records, is hidden until requested. "Disclosed" methods include putting a list or form into edit more, or an "add new" button to display the fields necessary to create new records. A disclosed interface is also appropriate when editing the data is an uncommon occurrence.

In general, viewing and editing data should happen within the same interface. The exceptions are complex multi-step forms, or data that merits an entirely focused, distraction-free view. In such cases, it may be appropriate to perform input/editing in a focus-stealing modal view.
